# Commit Convention

Use one of these prefixes:

- `feat:` new capability
- `fix:` bug fix
- `refactor:` internal restructuring without behavior change
- `docs:` documentation updates
- `test:` test changes
- `chore:` maintenance tasks

## Good Examples

- `feat: add word export endpoint`
- `fix: correct mileage parsing for LRS data`
- `docs: update release checklist`

## Bad Examples

- `update`
- `菫ｮ豁｣`
- `changes`
